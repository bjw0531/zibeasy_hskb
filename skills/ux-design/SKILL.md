# Skill: UX Philosophy

This project prioritizes a calm, intuitive, and psychologically comfortable user experience.

The goal is to create a UI/UX similar in feeling to:
- Toss
- Airbnb
- Apple-style interfaces

The interface must feel:
- simple
- clean
- calm
- friendly
- easy to understand immediately

Avoid interfaces that feel:
- cluttered
- technical
- crowded
- overwhelming
- visually noisy

---

## UX Core Principles

1. Reduce psychological burden
Users should not feel overwhelmed when entering a screen.

Prefer:
- progressive disclosure
- step-by-step flows
- minimal input fields visible at once

Avoid:
- large forms with many inputs at once
- dense information blocks

---

2. Visual breathing space

Spacing is extremely important.

Prefer:
- generous padding
- consistent margins
- clear separation between sections

Avoid:
- cramped layouts
- tightly packed elements

---

3. Highlight only what matters

Not everything should compete for attention.

The interface should clearly guide the user toward:
- the most important action
- the most important information

Use:
- hierarchy
- contrast
- spacing

Avoid:
- multiple strong focal points

---

4. Familiar patterns

Users should not need to learn new UI patterns.

Prefer interactions similar to:
- mobile apps
- well-known platforms

If a design choice may confuse users, prefer the simpler and more familiar option.

---

5. Low cognitive load

Users should be able to understand screens instantly.

Avoid:
- complicated filters
- excessive buttons
- unclear icons
- overly technical language

Prefer:
- clear labels
- simple controls
- readable layouts

---

## Interaction Tone

The product should feel:

friendly  
approachable  
calm  
modern  
not corporate or rigid

Avoid making the interface feel like:
- enterprise software
- government service portals
